# Adobe-Captivate_MS-Planner-Simulation
**Technical Stack**
Authoring Tool: Adobe Captivate (HTML5)

Methodology: Scenario-Based Learning (SBL)

Deployment: GitHub Pages  

**Project:**
Process Transformation via Microsoft Planner
Strategic Workforce Productivity & Project Management Solution

**Project Overview**
This project serves as a comprehensive case study in Process Transformation. It addresses the common organizational challenge of "app fatigue" and fragmented workflows. By analyzing the gap between inefficient and efficient productivity habits, this training provides a roadmap for utilizing Microsoft Planner as a central engine for project success.

**The Transformation Goal**
To transition a workforce from reactive, disorganized task-tracking to a proactive, standardized project management methodology that ensures accountability and visibility.

**Instructional Architecture**
The simulation is built on a "Transfer of Learning" framework, moving through four critical phases:

Process Analysis: Learners evaluate real-world examples of inefficient app usage (e.g., "The Email Trap") versus streamlined digital workflows.

Guided Simulation: A hands-on, risk-free environment where learners configure MS Planner buckets, labels, and dependencies.

Critical Assessment: Interactive knowledge checks focused on situational decision-making rather than just button-clicking.

Applied Project: A final capstone activity where learners must apply the transformation principles to a sample project to demonstrate mastery and skill transfer.
